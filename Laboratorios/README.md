# Laboratórios – 5 sprints

Entregas da disciplina **Laboratório de Experimentação de Software** (PUC Minas), organizadas em 5 sprints conforme o plano oficial:

**[Repositório da disciplina → LABORATORIOS](https://github.com/joaopauloaramuni/laboratorio-de-experimentacao-de-software/tree/main/LABORATORIOS)**

---

| Sprint | Pasta | Tema |
|--------|--------|------|
| **1** | [Lab-01](Lab-01/) | Características de repositórios populares (GitHub, com subpastas por etapa: Lab01S01, Lab01S02, etc.) |
| **2** | [Lab-02](Lab-02/) | Um estudo das características de qualidade de sistemas Java |
| **3** | [Lab-03](Lab-03/) | Caracterizando a atividade de code review no GitHub |
| **4** | [Lab-04](Lab-04/) | Visualização de dados utilizando uma ferramenta de BI |
| **5** | [Lab-05](Lab-05/) | GraphQL vs REST – Um experimento controlado |

Em cada pasta de laboratório estão (ou estarão) o enunciado em PDF, planejamento, scripts e relatórios da respectiva entrega. Para o Lab-01, cada etapa (S01, S02, S03) possui subpasta própria com scripts, dados e relatórios independentes.
